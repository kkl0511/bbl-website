# BBL 연구 노트 — 운영 가이드 (CMS 버전)

이 가이드는 **Decap CMS 관리자 페이지**에서 글을 올리는 방법을 안내합니다.

---

## 🎯 기본 개념

- 사이트 주소 뒤에 `/admin/` 을 붙이면 **관리자 페이지**가 열립니다 (예: `https://biomotion.co.kr/admin/`).
- 로그인하면 Word 같은 **글 에디터** 가 나타납니다.
- 글을 저장하면 GitHub에 자동 커밋되고, 1~2분 뒤 사이트에 자동 반영됩니다.
- **파일 편집이나 업로드는 이제 필요 없습니다.**

---

## 📝 새 글 작성하기

### 1. 관리자 페이지 열기

1. 브라우저 주소창에 `https://(교수님 사이트 주소)/admin/` 입력
2. 로그인 화면 → 초대받은 이메일과 비밀번호 입력
3. 관리자 대시보드 진입

### 2. 글 쓰기

1. 왼쪽 메뉴에서 **"연구 노트"** 클릭
2. 우측 상단 **"New 연구 노트"** 버튼 클릭
3. 필드 입력:

| 필드 | 설명 |
|------|------|
| **제목** | 글 제목 |
| **발행일** | 오늘 날짜 (자동 설정되지만 필요 시 변경) |
| **카테고리** | 드롭다운에서 선택 (공지/연구동향/바이오메카닉스 등) |
| **요약** | 2~3문장. 블로그 카드·SNS 공유 시 노출 |
| **작성자** | 기본값 "BBL 연구팀" (변경 가능) |
| **읽기 시간** | 예: "5분" |
| **대표 이미지** | 선택. 드래그 앤 드롭으로 업로드 가능 |
| **초안** | 체크하면 저장만 하고 사이트에 공개되지 않음 |
| **본문** | Markdown 에디터로 작성 |

### 3. 본문 편집기 사용법

툴바 버튼으로 서식을 지정할 수 있습니다:
- **굵게** / *기울임*
- 제목 (H2, H3)
- 인용구
- 불릿·번호 리스트
- 링크 삽입
- 이미지 삽입 (드래그 앤 드롭)
- 코드 블록

우측 패널에서 **실시간 미리보기**를 확인할 수 있습니다.

### 4. 저장 & 발행

- **Save**: 임시 저장 (사이트 반영 안 됨)
- **Publish Now**: 즉시 발행 → 1~2분 후 사이트에 표시됨
- 초안 체크박스를 켰다면 "Publish"를 눌러도 사이트에는 안 보입니다

---

## 📷 이미지 넣기

1. 본문 에디터에서 이미지 버튼 클릭 또는 드래그 앤 드롭
2. 이미지가 `assets/uploads/` 폴더에 자동 업로드됨
3. 본문에 자동 삽입됨

**권장 사양**: 가로 폭 1200px 이하 / 500KB 이하 / JPG·PNG·WebP

---

## ✏️ 글 수정 / 삭제

1. 관리자 페이지 → 연구 노트 목록에서 글 클릭
2. 수정 후 Publish 버튼
3. 삭제: 우측 상단 "Delete entry" 버튼

**모든 변경은 Git 히스토리에 기록됩니다.** 실수로 지워도 GitHub에서 복구 가능합니다.

---

## 👥 공동 편집자 초대

조교·대학원생·연구원을 편집자로 초대할 수 있습니다:

1. Netlify 대시보드 → Identity 탭
2. "Invite users" 버튼 → 이메일 입력
3. 초대받은 사람은 이메일 링크로 가입 → 곧바로 `/admin/` 접근 가능

---

## 🛠 문제 해결

### 로그인이 안 됩니다
- 초대 이메일의 링크를 **만료 전(24시간 이내)** 에 클릭했는지 확인
- 비밀번호 재설정: 로그인 화면의 "Forgot password?" 클릭

### 글을 저장했는데 사이트에 안 보입니다
- 초안 체크박스가 켜져있는지 확인
- 배포까지 1~3분 정도 걸릴 수 있음
- Netlify 대시보드 → Deploys 탭에서 **빌드 상태** 확인 (실패 시 빨간 메시지 표시)

### 관리자 페이지가 하얀 화면
- 브라우저 콘솔 열기 (F12) → 오류 메시지 확인
- 브라우저 캐시 지우고 새로고침 (Ctrl+Shift+R)

## 📚 글쓰기 팁

### 좋은 글의 구조
1. **리드**(1~2문단): 글 전체 요지를 짧게
2. **개념 설명**(2~4챕터)
3. **현장 적용 / 사례**
4. **마무리 + 다음 편 예고** (연재 형식으로 독자 재방문 유도)

### BBL 서비스 유입 전략
- 본문 안에서는 홍보 자제, "BBL의 측정에서는..." 같은 **경험 공유 톤**
- 글 하단 CTA 박스에서 자연스럽게 상담 신청 유도 (자동 배치됨)

### 톤·길이
- 5~8분 분량이 가장 잘 읽힘
- 전문 용어는 첫 등장 시 **한글 + 영문 + 단위** 함께 표기
- 예: "팔꿈치 내반 토크(elbow valgus torque, N·m)"

### 주의사항
- 실제 선수 이름·소속 특정 피하기 (A 구단 좌완 투수 정도까지)
- 논문 인용 시 저자·연도 밝히되 원문 그대로 옮기지 않고 풀어 설명
- 이미지는 자체 제작 또는 CC0 라이선스만 사용

## 🔧 기술 구조 (참고용)

- 글 파일은 `posts/*.md` 에 저장 (Markdown + frontmatter)
- 빌드 시 `build-index.js` 가 `posts/` 를 스캔해 `posts.js` (글 목록)를 자동 생성
- `post.html` 이 URL의 `?slug=xxx` 를 읽어 해당 Markdown을 표시
- CMS 설정은 `admin/config.yml` 에 있음
- 카테고리 추가·수정은 `admin/config.yml` 의 `options` 부분을 고친 뒤 git push

의심스러울 땐 건드리지 마시고 말씀해 주세요.
